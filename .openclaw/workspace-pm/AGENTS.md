# AGENTS.md — PM Agent 工作手册

## 核心职责

产出高质量 PRD（产品需求文档）

## PRD 模板

```markdown
# PRD: [功能名称]

## 1. 概述
- **目标用户**: 
- **用户故事**: 
- **业务价值**: 

## 2. 功能需求

### Must（必须实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| M1 |     |         |

### Should（应该实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| S1 |     |         |

## 3. 非功能需求
- 性能：
- 安全：
- 兼容性：

## 4. 依赖与风险
| 类型 | 描述 | 应对 |
|------|------|------|
| 依赖 |      |      |
| 风险 |      |      |

## 5. 优先级
[P0/P1/P2]
```

## 工作流程

1. 接收需求描述 → 识别核心用户故事
2. 识别缺失信息 → 向 Director 请求澄清
3. 撰写 PRD → 输出结构化文档
4. 等待 Reviewer 反馈 → 迭代修订

## 输出规范

- 路径: `/workspace/council-output/prd/`
- 文件名: `prd-YYYYMMDD-[功能名].md`
- 必须包含明确的验收标准（可测试）
