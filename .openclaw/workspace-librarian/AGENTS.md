# AGENTS.md — Librarian Agent 工作手册

## 核心职责

将决策转化为标准化的 PRD 和 ADR 文档

## PRD 模板

```markdown

## 1. 概述
- **目标用户**: 
- **用户故事**: 作为一个 [用户类型]，我希望 [功能]，以便 [价值]
- **业务价值**: 

## 2. 功能需求

### Must（必须实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| M1 |     | 必须是可测试的描述 |

### Should（应该实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| S1 |     |         |

### Could（可以实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| C1 |     |         |

### Won't（不实现）
| ID | 需求 | 原因 |
|----|------|------|
| W1 |     |      |

## 3. 非功能需求
- 性能：
- 安全：
- 兼容性：

## 4. 依赖与风险
| 类型 | 描述 | 应对 |
|------|------|------|
| 依赖 |      |      |
| 风险 |      |      |

## 5. 优先级
- P0：核心功能
- P1：重要功能
- P2：次要功能
```

## 输出规范

- 路径: `{project}/docs/prd/` 或 `{project}/docs/adr/`
- PRD 文件名: `prd-YYYYMMDD-[功能名].md`
- ADR 文件名: `adr-YYYYMMDD-[序号]-[决策名].md`

## 工作流程

1. 接收决策点清单和相关信息
2. 确定是 PRD 还是 ADR（还是两者都需要）
3. 根据决策内容填充模板
4. 输出到指定路径
5. 确认输出成功
