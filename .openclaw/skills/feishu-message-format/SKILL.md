# 飞书消息格式化技能

## 描述
自动根据消息内容类型选择最佳飞书消息格式（text/post），优化手机端阅读体验。

## 触发条件
- 用户提到"飞书消息格式"、"消息渲染"、"卡片消息"
- 使用 `/format` 命令手动切换
- 发送任务汇报/代码审查/架构评审时自动应用

## 使用方式

### 自动模式（默认）
技能自动检测内容特征选择格式：

**触发 post 卡片的场景**：
- ✅ 任务进度汇报（含"进度"、"完成"、"下一步"等关键词）
- ✅ 代码审查结果（含"✅"、"❌"、"审查"等关键词）
- ✅ 架构评审摘要（含"架构"、"设计"、"评审"等关键词）
- ✅ 消息 ≥ 10 行 且 包含结构化内容

**保持纯文本的场景**：
- 日常对话
- 简短确认（< 5 行）
- 快速回复

### 手动模式（命令切换）
用户指定格式：
- `/format card` — 强制卡片格式（下一条消息）
- `/format text` — 强制纯文本（下一条消息）
- `/format auto` — 恢复自动选择（默认）

## 格式选择规则

### 使用纯文本 (type=1) 当：
- 消息 < 5 行
- 纯对话/简短回复
- 无代码块/表格/复杂格式

### 使用富文本卡片 (type=2) 当：
- 消息 ≥ 5 行
- 包含代码块 (```)
- 包含表格 (| 列 |)
- 包含状态标记 (✅❌⚠️🔴🟡)
- 包含多级标题 (## )
- 任务汇报/代码审查/进度更新

## @mention 处理

自动转换 `<at user_id="xxx">name</at>` 为飞书原生 @ 格式。

## 手机端优化

1. 卡片启用 `wide_screen_mode: true`
2. 用分隔线 `---` 分隔大块内容
3. 粗体标记关键信息 `**重点**`
4. 代码块超过 20 行建议发文件链接
5. 适量使用表情符号增强视觉层次

## 示例

### 输入（Markdown）
```markdown
## 任务进度

**项目**: PTX-EMU
**状态**: ✅ 编译通过

### 完成项
- [x] CUDA kernel 优化
- [x] 内存管理重构

### 下一步
- 性能测试
```

### 输出（飞书 post 卡片）
```json
{
  "config": { "wide_screen_mode": true },
  "elements": [
    { "tag": "div", "text": { "content": "**任务进度**", "tag": "lark_md" } },
    { "tag": "hr" },
    { "tag": "div", "text": { "content": "**项目**: PTX-EMU\n**状态**: ✅ 编译通过", "tag": "lark_md" } },
    { "tag": "note", "elements": [{ "tag": "plain_text", "content": "下一步：性能测试" }] }
  ]
}
```
