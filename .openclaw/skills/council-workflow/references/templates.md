# PRD/ADR 模板参考

## PRD 模板

```markdown
# PRD: [功能名称]

## 1. 概述
- **目标用户**: [描述目标用户群体]
- **用户故事**: 作为一个 [用户类型]，我希望 [功能]，以便 [价值]
- **业务价值**: [描述业务价值]

## 2. 功能需求

### Must（必须实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| M1 |     |         |

### Should（应该实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| S1 |     |         |

### Could（可以实现）
| ID | 需求 | 验收标准 |
|----|------|---------|
| C1 |     |         |

### Won't（不实现）
| ID | 需求 | 原因 |
|----|------|------|
| W1 |     |      |

## 3. 非功能需求
- 性能：[具体指标]
- 安全：[安全要求]
- 兼容性：[兼容性要求]
- 可用性：[可用性要求]

## 4. 依赖与风险
| 类型 | 描述 | 应对 |
|------|------|------|
| 依赖 |      |      |
| 风险 |      |      |

## 5. 优先级
- P0：[核心功能]
- P1：[重要功能]
- P2：[次要功能]
```
