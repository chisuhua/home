## 🎯 五大核心原则

### 1️⃣ 架构先行
> **在开始编码前，必须先完成架构设计和接口定义。**

✅ 检查清单:
- [ ] 是否已阅读相关现有代码？
- [ ] 是否有架构图/模块划分图？
- [ ] 是否定义了清晰的模块接口？
- [ ] 是否制定了实施计划？

---

### 2️⃣ 编译质量
> **每次提交前必须编译通过，零警告策略。**

✅ 检查清单:
- [ ] 编译是否无错误？
- [ ] 编译是否无警告？
- [ ] 是否使用了正确的 IO 访问模式？
- [ ] 类型是否匹配？

---

### 3️⃣ 问题解决
> **系统性分析问题，根因导向解决，避免表面修复。**

✅ 检查清单:
- [ ] 是否已完整收集错误信息？
- [ ] 是否已分类错误类型？
- [ ] 是否已找到根本原因？
- [ ] 解决方案是否系统性？

---

### 4️⃣ 技能复用
> **每次解决问题后，必须考虑是否可以创建/复用技能。**

✅ 检查清单:
- [ ] 是否搜索了现有技能？
- [ ] 问题是否具有通用性？
- [ ] 是否可以创建新技能？
- [ ] 技能是否已文档化？

---

### 5️⃣ 流程驱动
> **所有复杂任务必须通过 acf-workflow 流程驱动。**

✅ 检查清单:
- [ ] 任务是否已分解？
- [ ] ACP 会话是否已创建？
- [ ] 任务说明是否清晰？
- [ ] 验收标准是否明确？
