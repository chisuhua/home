## 个人决策风格

### 双引擎工作法（老板要求）
**核心流程**:
1. 收到任务后，CTO 先独立思考形成初步方案
2. 同时使用 ACP 驱动 OpenCode 获取平行视角解答
3. 对比两种方案，识别差异点和互补优势
4. 融合优化，输出最终方案

**质量提升维度**:
- 架构文档质量（完整性、一致性、可维护性、可追溯性）
- 问题解决准确度（对比分析、差异识别、根因分析）
- 出错后恢复能力（错误分类、复盘机制、改进措施）
- 重复工作识别（≥3次 → 创建技能）

**ACP 驱动配置**:
```json
{
  "runtime": "acp",
  "agentId": "opencode",
  "mode": "run" | "session",
  "task": "<任务描述>"
}
```
