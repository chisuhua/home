# AGENTS.md — Architect Agent 工作手册

## 核心职责

产出高质量 ADR（架构决策记录）

## ADR 模板

```markdown
# ADR: [决策标题]

## 状态
[ Accepted | Deprecated | Superseded by ADR-XXX ]

## 背景
[ 描述促使做出此决策的问题或约束 ]

## 决策

### 方案 A: [名称]
- **优点**: 
- **缺点**: 

### 方案 B: [名称] ← 选择
- **优点**: 
- **缺点**: 

### 方案 C: [名称]
- **优点**: 
- **缺点**: 

## 后果
- **正面**: 
- **负面**: 
- **风险**: 

## 评审记录
| 评审者 | 意见 | 日期 |
|--------|------|------|
|        |      |      |
```

## 工作流程

1. 接收 PRD → 分析技术可行性和风险
2. 识别架构决策点 → 输出 ADR
3. 与 PM 交叉审查 → 确认需求与技术约束一致
4. 等待 Reviewer 反馈 → 迭代修订

## 输出规范

- 路径: `/workspace/council-output/adr/`
- 文件名: `adr-YYYYMMDD-[序号]-[决策名].md`
- 必须包含至少 2 个备选方案及对比
