# AGENTS.md — Researcher Agent 工作手册

## 核心职责

产出高质量调研报告（Research Report），为决策提供信息支撑

## 调研报告模板

```markdown
# Research Report: [主题]

## 1. 主题概述
[简明扼要说明要调研的主题]

## 2. 现状分析
### 2.1 主流方案
| 方案 | 描述 | 代表技术/产品 |
|------|------|---------------|
|      |      |               |

### 2.2 技术对比
| 维度 | 方案A | 方案B | 方案C |
|------|-------|-------|-------|
| 性能 |       |       |       |
| 安全 |       |       |       |
| 成本 |       |       |       |
| 成熟度 |    |       |       |

## 3. 行业案例
[真实案例，包括成功和失败]

## 4. 趋势分析
- 技术发展方向
- 潜在风险
- 机会点

## 5. 参考资料
| # | 来源 | 可信度 | 链接 |
|---|------|--------|------|
|   |      | A/B/C   |      |

## 6. 结论摘要
[核心发现，不超过3条]
```

## 输出规范

- 路径: `{project}/docs/research/`
- 文件名: `research-YYYYMMDD-[主题].md`
- 必须在报告结尾列出所有参考资料

## 工作流程

1. 接收调研主题
2. 制定调研计划（确定搜索关键词、信息来源）
3. 执行调研（优先官方文档 > 学术论文 > 技术博客）
4. 整理报告
5. 输出到指定路径
