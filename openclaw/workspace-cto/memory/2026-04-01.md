## CppHDL 编译错误修复

### 问题 1: operators.h 位移操作未定义行为

**错误信息**:
```
/workspace/CppHDL/include/core/operators.h:758:23: error: 
right operand of shift expression '(1 << 32)' is greater than or equal 
to the precision 32 of the left operand [-fpermissive]
```

**根因分析**:
- `operator<<` 中计算 `rhs_extra_width` 时使用 `(1 << ch_width_v<RHS>) - 1`
- 当 `ch_width_v<RHS> >= 32` 时，`1 << 32` 触发未定义行为（UB）
- `1` 是 `int` 类型（32 位），位移量 >= 32 超出类型精度

**修复方案**:
参考 `include/core/literal.h` 第 33 行的安全实现：
```cpp
// 方案 1: 使用 64 位类型（推荐）
return (static_cast<uint64_t>(1) << ch_width_v<RHS>) - 1;

// 方案 2: 添加边界检查
if constexpr (width >= 64) {
    return ~0u;  // 最大值
} else {
    return (1u << width) - 1;
}
```

**已创建技能**: `/workspace/CppHDL/skills/cpphdl-shift-fix/SKILL.md`

### 问题 2: context.tpp 断言宏问题

**错误信息**:
```
!(condition) && !ch::detail::in_static_destruction()) { \
```

**根因**: 断言宏在静态析构期间的行为未正确处理

**修复方向**: 参考 `CHREQUIRE` 宏实现，添加静态析构检查

### 其他发现的潜在风险点

通过 `grep -rn "1 <<" include/` 发现以下位置需要检查：
- `include/chlib/fifo.h:30` - `MAX_COUNT = (1 << (N - 1))`
- `include/chlib/fifo.h` 多处 - `(1 << ADDR_WIDTH)`
- `include/chlib/axi4lite.h:116` - `(1 << ADDR_WIDTH)`

**建议**: 对这些位置添加边界检查或使用 `1ULL` 前缀

## 经验教训

1. **位移操作必须明确类型**: 永远不要写 `1 << N`，要写 `1u << N` 或 `1ULL << N`
2. **模板元编程中的边界检查**: 在 `if constexpr` 中处理极端情况（0, 32, 64）
3. **参考现有安全实现**: `literal.h` 已经有正确的位移安全处理代码，应该复用
