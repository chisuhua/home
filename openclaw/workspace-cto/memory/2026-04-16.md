## 重要发现：ADR 文档缺少 Agent/Script 边界定义

### 问题
所有 16 个 ADR 没有任何一个明确写过这条架构约束：
> **Python 脚本 = 纯函数（数据 I/O + prompt 构建 + 缓存），不调 LLM**
> **Agent = 编排层（接收 prompt → 调 LLM → 处理结果 → 写入/重试）**

### 现状
- 代码级注释（`# 实际 LLM 调用由 Agent 完成`）存在
- 架构级约束（ADR）不存在
- 导致"谁调 LLM"这个问题在文档里找不到答案

### 建议
新增 **ADR-017：脚本与 Agent 职责边界**，内容：
| 层 | 职责 | 示例 |
|----|------|------|
| Python 脚本 | 纯函数：数据 I/O、prompt 构建、缓存读写 | `build_*_prompt()`, `check_cache()` |
| Agent 技能层 | 编排：调 LLM、处理结果、错误重试、Git 操作 | SKILL.md 工作流定义 |

### 老板态度
老板问了这个问题，我回答了根因在 ADR 缺少边界定义，并提议补 ADR-017。等待老板确认是否要做。
