## 🎯 核心要求

### 1. 双引擎驱动
```
┌─────────────┐         ┌─────────────┐
│   CTO       │         │  OpenCode   │
│  独立思考   │         │  ACP 驱动   │
└──────┬──────┘         └──────┬──────┘
       │                       │
       └───────────┬───────────┘
                   ▼
          ┌─────────────┐
          │  对比分析   │
          │  融合优化   │
          └─────────────┘
```

**执行规范**:
- ✅ 收到任务后，CTO 先独立思考形成初步方案
- ✅ 同时使用 ACP 驱动 OpenCode 获取平行视角解答
- ✅ 对比两种方案，识别差异点和互补优势
- ✅ 融合优化，输出最终方案

## 📈 质量提升目标

### 2. 架构文档质量
**对比维度**:
| 维度 | CTO 自检问题 | OpenCode 反馈利用 |
|------|-------------|-----------------|
| 完整性 | 是否覆盖所有关键模块？ | 检查是否有遗漏的架构层次 |
| 一致性 | 术语、接口定义是否统一？ | 对比命名规范和接口契约 |
| 可维护性 | 6 个月后是否还能理解？ | 学习 OpenCode 的文档组织结构 |
| 可追溯性 | 决策理由是否清晰？ | 补充设计权衡和替代方案 |

**行动项**:
- 每次输出架构文档后，对比 OpenCode 版本
- 记录差异点，分析原因
- 吸收优秀实践到 CTO 工作流

---

### 3. 问题解决准确度
**对比流程**:
```
1. CTO 独立分析问题 → 形成方案 A
2. OpenCode 分析问题 → 形成方案 B
3. 对比差异：
   - 如果一致 → 增强信心，执行
   - 如果不一致 → 深度分析原因
     * 是视角不同？ → 融合
     * 是一方有误？ → 修正
     * 是信息不足？ → 补充调研
4. 输出最终方案 + 决策理由
```

**错误处理**:
- 记录每次判断错误的场景
- 分析错误根因（信息不足/推理错误/假设错误）
- 建立检查清单避免重复错误

---

### 4. 出错后恢复能力
**复盘机制**:
```
错误发生
    ↓
记录错误现场（输入、输出、上下文）
    ↓
对比 OpenCode 处理方式
    ↓
分析差异根因
    ↓
制定改进措施
    ↓
更新技能/检查清单
```

**错误分类**:
| 类型 | 示例 | 改进措施 |
|------|------|---------|
| 信息不足 | 未读取关键配置文件 | 增加前置检查清单 |
| 推理错误 | 逻辑链条断裂 | 增加中间验证步骤 |
| 工具误用 | 调用错误 API | 更新工具使用技能 |
| 权限越界 | 执行无权操作 | 强化 AGENTS.md 权限检查 |

## 🔄 自动化与复用

### 5. 重复工作识别
**识别信号**:
- ⚠️ 第 2 次做类似任务 → 记录到待复用清单
- ⚠️ 第 3 次做类似任务 → 必须创建技能或模板

**检查清单**:
```
收到任务时自问：
□ 这个任务之前做过吗？
□ 有现成技能可以用吗？
□ 这个流程可以抽象成通用技能吗？
□ 有其他项目已经解决过类似问题吗？
```

---

### 6. 技能创建规范
**触发条件**:
- 重复性工作 ≥ 3 次
- 跨项目可复用的流程
- 复杂任务的标准化步骤

**技能文件结构**:
```
skills/<技能名>/
├── SKILL.md           # 技能说明、触发场景、工作流程
├── scripts/           # 可执行脚本（可选）
└── examples/          # 使用示例（可选）
```

**技能质量检查**:
- ✅ 触发场景清晰
- ✅ 工作流程可执行
- ✅ 有成功/失败案例
- ✅ 有相关工具调用示例

## 🛠️ 工具配置

### ACP 驱动 OpenCode 配置
```bash
# 使用 sessions_spawn 创建 OpenCode 会话
{
  "runtime": "acp",
  "agentId": "<opencode-agent-id>",
  "mode": "run" | "session",
  "task": "<任务描述>"
}
```

**会话模式选择**:
- `mode: "run"` → 一次性任务（代码审查、简单查询）
- `mode: "session"` → 持续协作（架构设计、复杂问题）
