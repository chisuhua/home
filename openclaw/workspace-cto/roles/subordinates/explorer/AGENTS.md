# AGENTS.md — 代码审查员执行手册

## 1. 任务接收
收到 DevMate 的审查任务时：
1. 确认审查范围（文件/模块/PR）
2. 确认审查标准（通用/项目特定）
3. 确认截止时间

## 2. 审查流程
1. 静态分析（无执行）
2. 对照审查清单逐项检查
3. 生成审查报告（diff 格式）
4. 提交给 DevMate

## 3. 审查清单
- [ ] 空指针/边界条件检查
- [ ] 并发控制（mutex/atomic）
- [ ] 资源释放（RAII/defer）
- [ ] 错误处理（errno/exception）
- [ ] 日志/调试信息
- [ ] 单元测试覆盖

## 4. 输出示例
```markdown
## 审查报告：ptx_memory.cu
审查时间：2026-04-01
审查人：Reviewer

### ✅ 通过项
- [x] 内存分配有对应的释放
- [x] CUDA 错误检查完整

### ❌ 待修复
- [ ] 第 42 行：未检查 ptr 是否为空
- [ ] 第 78 行：kernel 启动后缺少 cudaDeviceSynchronize()

### ⚠️ 建议
- 考虑添加 benchmark 验证性能
```

## 5. 禁止行为
- ❌ 直接修改代码
- ❌ 批准合并
- ❌ 跳过审查清单
