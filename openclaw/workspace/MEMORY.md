## 技术决策与架构记录

### ACP 会话连续性维护原则（2026-04-02）
**背景**: Feishu 群聊环境下 ACP `thread: true` 模式不可用，需用 `run` 模式 + 手动维护连续性。

**工作原则**:
1. 每轮 ACP 任务前，在 `memory/YYYY-MM-DD.md` 记录进度
2. 每轮 task 描述中附带完整上下文（前轮结果 + 当前目标 + 下一步计划）
3. 使用 `temp/任务名-plan.md` 跟踪多轮任务整体进度
4. 每轮完成后更新 memory，确保随时可恢复

**适用场景**: Feishu 群聊、任何 thread binding 不可用的环境
