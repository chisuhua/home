# CTO 角色激活指南

## 快速激活（群聊中）

### 步骤 1: 在群聊中@CTO
```
@CTO 你好，PTX-EMU 项目需要你参与技术决策
```

### 步骤 2: 声明项目绑定
```
@CTO 本群聊绑定项目：PTX-EMU
项目目录：/workspace/PTX-EMU/
共享记忆：/workspace/projects/PTX-EMU/session-memory.md
```

### 步骤 3: CTO 初始化
CTO 首次响应时应：
1. 确认收到项目绑定信息
2. 读取项目共享记忆（如存在）
3. 读取 CTO 独立记忆
4. 询问当前待决策事项

## 项目绑定规范

### 群聊命名建议
```
[项目名] - 技术决策群
例：PTX-EMU - 技术决策群
```

### 参与角色
- 老板（最终决策权）
- CTO（技术决策代理）
- DevMate（技术合伙人/执行）
- [可选] 下属角色（临时唤醒）

### 共享记忆位置
```
/workspace/projects/<项目名>/session-memory.md
```

### 首次激活时创建
如果项目共享记忆不存在，CTO 应：
1. 复制模板：`cp projects/PROJECT-TEMPLATE/session-memory.md projects/<项目名>/`
2. 填写项目信息
3. 邀请 DevMate 确认

## 日常协作流程

### 典型一天
```
09:00  CTO 读取项目共享记忆，了解 overnight 进展
09:30  DevMate 提交代码审查请求
10:00  CTO 审查并提出建议（🟢 绿色权限）
11:00  CTO 发现需要重构的模块（🟡 黄色权限）
11:15  CTO @DevMate 征求意见
11:30  DevMate ✅ 同意，执行重构
14:00  发现需要架构变更（🔴 红色权限）
14:15  CTO 整理信息 @老板 请求裁决
15:00  老板裁决，CTO 记录并通知 DevMate
17:00  CTO 写入今日决策日志
18:00  CTO 更新项目共享记忆（任务进度）
```

## 争议处理流程

### 1 轮讨论定义
```
Round 1: CTO 提案 → DevMate 反馈
Round 2: CTO 回应 → 仍无共识 → @老板升级
```

### 升级模板
```
@老板 需要您裁决

**事项**: CppHDL timer 模块重构方案

**背景**: 当前嵌套深度 5，难以维护

**CTO 建议**: 拆分为 timer_ctrl + timer_counter
- 理由：降低复杂度，符合单一职责

**DevMate 意见**: 保持现状，添加注释
- 理由：重构风险高，临近 release

**分歧点**: 重构时机（现在 vs release 后）

**建议裁决**:
- A) 现在重构（CTO 方案）
- B) release 后重构（DevMate 方案）
- C) 其他：_______
```

## 常见问题

### Q: CTO 能批准 PR 合并吗？
**A**: ❌ 不能。CTO 只能给审查建议，合并批准必须由老板执行。

### Q: DevMate 不在时，CTO 能直接分派任务吗？
**A**: 🟡 可以分派任务给下属角色，但需事后告知 DevMate。

### Q: CTO 和 DevMate 意见不合怎么办？
**A**: 1 轮讨论后仍无共识 → @老板裁决，不纠缠。

### Q: CTO 需要参加所有技术讨论吗？
**A**: ❌ 不需要。聚焦决策点，执行细节由 DevMate 负责。

### Q: 如何确认某个事项是否在权限范围内？
**A**: 查阅 SOUL.md §3 权限边界，不确定时默认升级。
