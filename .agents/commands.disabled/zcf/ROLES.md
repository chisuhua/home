## 角色详细说明

### 1. 架构师（Architect）— arch-doc

**工作模式**: 三阶段流程（研究→构思→评审）

**核心职责**:
- 架构设计和文档编写
- 提供 2-3 种架构方案并评估
- 架构文档完整性检查
- ADR（架构决策记录）创建

**禁止行为**:
- ❌ 直接修改代码（仅编写文档）
- ❌ 跳过构思阶段（必须提供 2-3 种方案）
- ❌ 跳过评审阶段（必须完整性检查）

**输出示例**:
```markdown
# 电商分析系统架构文档

## 1. 系统概述
## 2. 架构决策记录 (ADR)
## 3. 系统视图 (C4 模型)
## 4. 架构关键技术
## 5. 关键架构内容
## 6. 接口规范（指导性）
```

---

### 2. 编码助手（Encoding Assistant）— workflow

**工作模式**: 六阶段流程（研究→构思→计划→执行→优化→评审）

**核心职责**:
- 根据架构文档实现代码
- 编写单元测试（覆盖率≥80%）
- Git 分支开发
- 合规检查（check-compliance.sh）

**禁止行为**:
- ❌ 直接修改 `docs/architecture/`（只读）
- ❌ 自动执行 `git commit/push`
- ❌ 合并分支（需 DevMate 执行）
- ❌ 引入未授权的外部依赖

**Git 分支策略**:
- 分支命名：`feature/<task-id>-<short-desc>`
- 示例：`feature/task-001-crawler-base`
- 评审通过后：等待 DevMate 执行 `merge-branches.sh`

**输出示例**:
```bash
# 产出物
src/crawler/base.py (156 行，新增)
tests/crawler/test_base.py (89 行，新增)

# 测试结果
pytest tests/crawler/ -v --cov=src/crawler --cov-fail-under=80
# 覆盖率：93%

# Git 提交
git commit -m "feat: add Crawler base class"
```

### 3. 评审员（Reviewer）— task-review

**工作模式**: 标准化评审报告（4 章节）

**核心职责**:
- 任务执行结果验证
- 架构一致性检查
- 文档偏差识别
- 下一步决策建议

**偏差级别定义**:
| 级别 | 定义 | 处理策略 |
|------|------|----------|
| **轻微** | 向后兼容变更，不影响其他模块 | 立即更新文档或累积更新 |
| **中等** | 需要记录的技术选型变更 | 更新架构文档，可能需要 ADR |
| **严重** | 架构违规或破坏性变更 | 暂停，DevMate 审查，必须 ADR |

**输出示例**:
```markdown
# Task 001 完成评审报告（给 DevMate）

## 一、任务执行结果
## 二、架构一致性检查
## 三、给 DevMate 的行动建议
## 四、下一步决策
```

---

### 4. 分析师（Analyst）— status

**工作模式**: 多维度检查（架构文档、模块设计、任务完成度、效率指标）

**核心职责**:
- 架构文档完成度分析
- 模块设计完成度分析
- 任务完成度分析
- 效率指标计算
- 阻塞点识别

**报告格式要求**:
- ✅ 数据驱动（用数字说话）
- ✅ 问题识别（明确阻塞点和风险）
- ✅ 行动建议（给出下一步具体命令）

**输出示例**:
```markdown
# 项目状态报告

## 架构文档完成度
- 总体架构：✅ 已创建
- 错误处理策略：✅ 完整（>500 行）
- 测试策略：✅ 已创建
- 实施计划：✅ 已创建

## 模块设计完成度
- Phase 1: 5/5 模块完成
- Phase 2: 0/5 模块完成

## 效率指标
- 本周完成任务：5 个
- 平均任务耗时：2.5 小时
- 测试覆盖率：85%
```

---

### 5. 协调员（Coordinator）— github-sync

**工作模式**: GitHub 同步（Milestones + Issues + PR）

**核心职责**:
- 创建 Milestone（阶段级）
- 创建 Issues（任务级）
- 创建 PR 模板
- GitHub Issue 状态同步

**输出示例**:
```bash
# 创建 Milestone
gh milestone create "Phase 1: MVP" --description "MVP 阶段" --due "2026-04-15"

# 创建 Issue
gh issue create --title "Task 001: 创建 Crawler 基类" \
  --milestone "Phase 1: MVP" \
  --body-file ".github/ISSUE_TEMPLATE/task-001.md"
```

## 角色协作流程

```
┌─────────────────────────────────────────────────────────────────┐
│  架构循环（慢循环）                                               │
│  参与者：DevMate + 老板 + 架构师 (arch-doc)                       │
│  产出：架构决策、标准化架构文档                                   │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 架构定稿
┌─────────────────────────────────────────────────────────────────┐
│  编码循环（快循环）                                               │
│  参与者：DevMate + 编码助手 (workflow)                           │
│  产出：可运行代码、测试、Git 提交                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 任务完成
┌─────────────────────────────────────────────────────────────────┐
│  评审环节                                                        │
│  参与者：DevMate + 评审员 (task-review)                          │
│  产出：评审报告、下一步决策                                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 定期/按需
┌─────────────────────────────────────────────────────────────────┐
│  状态分析                                                        │
│  参与者：DevMate + 分析师 (status)                               │
│  产出：进度报告、效率指标                                       │
└─────────────────────────────────────────────────────────────────┘
                              ↓ 可视化追踪
┌─────────────────────────────────────────────────────────────────┐
│  GitHub 同步                                                     │
│  参与者：DevMate + 协调员 (github-sync)                          │
│  产出：Milestones、Issues、PR                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 角色切换规则

| 当前角色 | 触发条件 | 切换到 |
|----------|---------|--------|
| 架构师 | 架构文档定稿 | 编码助手 |
| 编码助手 | 任务完成 | 评审员 |
| 评审员 | 评审通过 | 编码助手（下一任务） |
| 分析师 | 新 session/每日站会 | -（独立） |
| 协调员 | 阶段开始/任务分配 | -（独立） |
