## 1. 错误分类标准

### 1.1 错误级别定义

| 级别 | 名称 | 说明 | 响应时间 | 通知方式 | 示例 |
|------|------|------|----------|----------|------|
| **P0** | 致命错误 | 系统崩溃、数据丢失 | 立即 | 电话 + 短信 | 数据库连接失败、内存溢出 |
| **P1** | 严重错误 | 核心功能不可用 | 1 小时 | 短信 + 邮件 | API 服务宕机、支付失败 |
| **P2** | 一般错误 | 部分功能受影响 | 24 小时 | 邮件 | 单个模块异常、性能下降 |
| **P3** | 轻微错误 | 不影响核心功能 | 3 天 | 工单系统 | UI 显示问题、日志异常 |
| **P4** | 建议改进 | 非错误，改进建议 | 下次迭代 | 工单系统 | 性能优化、代码重构 |

### 1.2 错误码规范

**错误码格式**：
```
{MODULE}_{ERROR_TYPE}_{SPECIFIC_ERROR}
```

**项目错误码分配**：
| 模块 | 错误码前缀 | 范围 | 说明 |
|------|-----------|------|------|
| 通用模块 | `COMMON_` | 000-099 | 跨模块通用错误 |
| {{MODULE_1}} | `{{PREFIX_1}}_` | 100-199 | {{说明}} |
| {{MODULE_2}} | `{{PREFIX_2}}_` | 200-299 | {{说明}} |

**示例错误码**：
```
COMMON_INVALID_ARGUMENT
COMMON_TIMEOUT
{{PREFIX_1}}_CONNECTION_LOST
{{PREFIX_2}}_MAX_RETRIES_EXCEEDED
```

## 2. 错误处理模式

### 2.1 重试模式（Retry Pattern）

**适用场景**：
- 网络请求超时
- 数据库连接暂时失败
- 第三方服务临时不可用

**实现要求**：
```python
# 必须包含：
# 1. 最大重试次数（默认 3 次）
# 2. 退避策略（指数退避：1s, 2s, 4s, 8s...）
# 3. 最大延迟限制（避免无限等待）
# 4. 重试失败后抛出明确异常
```

**配置示例**：
```yaml
retry:
  max_attempts: 3
  initial_delay: 1.0  # 秒
  backoff_multiplier: 2.0
  max_delay: 60.0  # 秒
```

---

### 2.2 熔断模式（Circuit Breaker Pattern）

**适用场景**：
- 第三方服务持续失败
- 防止级联故障
- 保护系统资源

**状态转换**：
```
CLOSED（正常） → OPEN（熔断） → HALF_OPEN（探测） → CLOSED（恢复）
```

**触发条件**：
- 失败率 > 50%（1 分钟内）
- 慢调用 > 3 秒（1 分钟内）
- 连续失败次数 > 5 次

**实现要求**：
```python
# 必须包含：
# 1. 失败计数器（滑动窗口统计）
# 2. 熔断状态机（CLOSED/OPEN/HALF_OPEN）
# 3. 自动恢复机制（HALF_OPEN 状态探测）
# 4. 熔断事件日志记录
```

---

### 2.3 降级模式（Fallback Pattern）

**适用场景**：
- 非核心功能失败
- 缓存数据可用
- 返回默认值

**降级策略**：
1. **缓存降级**：返回最近一次成功结果
2. **默认值降级**：返回安全默认值
3. **静态页面降级**：返回预渲染页面
4. **功能关闭**：隐藏失败功能入口

**实现要求**：
```python
# 必须包含：
# 1. 降级条件判断（什么情况下触发降级）
# 2. 降级数据源（缓存/默认值/配置）
# 3. 降级事件日志（记录降级发生）
# 4. 降级恢复检测（自动恢复正常流程）
```

---

### 2.4 超时控制（Timeout Pattern）

**适用场景**：
- 外部 API 调用
- 数据库查询
- 文件 IO 操作

**超时标准**：
| 操作类型 | 超时时间 | 处理策略 |
|---------|---------|---------|
| HTTP 请求 | 30 秒 | 重试 2 次后失败 |
| 数据库查询 | 10 秒 | 重试 1 次后失败 |
| 文件 IO | 5 秒 | 直接失败 |
| 内存计算 | 60 秒 | 直接失败 |

**实现要求**：
```python
# 必须包含：
# 1. 操作级别超时设置
# 2. 超时异常捕获和转换
# 3. 资源清理（关闭连接/释放锁）
# 4. 超时事件日志
```

## 3. 错误日志记录

### 3.1 日志级别

| 错误级别 | 日志级别 | 记录内容 |
|---------|---------|---------|
| P0 | ERROR | 完整堆栈 + 上下文 + 告警 |
| P1 | ERROR | 完整堆栈 + 上下文 |
| P2 | WARN | 错误信息 + 关键参数 |
| P3 | INFO | 错误信息摘要 |
| P4 | DEBUG | 调试信息 |

### 3.2 日志格式

**标准格式**：
```json
{
  "timestamp": "2026-03-26T10:30:00Z",
  "level": "ERROR",
  "error_code": "CRAWLER_TIMEOUT",
  "error_level": "P2",
  "module": "crawler",
  "message": "请求超时",
  "stack_trace": "...",
  "context": {
    "url": "https://example.com",
    "timeout": 30,
    "attempt": 2
  },
  "user_id": "user_123",
  "request_id": "req_456"
}
```

**必填字段**：
- `timestamp`: ISO 8601 时间戳
- `level`: 日志级别（ERROR/WARN/INFO/DEBUG）
- `error_code`: 错误码
- `error_level`: 错误级别（P0-P4）
- `module`: 模块名称
- `message`: 错误描述
- `context`: 错误上下文（键值对）

## 4. 错误监控与告警

### 4.1 监控指标

**核心指标**：
- 错误率（错误数 / 总请求数）
- P0/P1 错误数量（实时）
- 平均恢复时间（MTTR）
- 熔断触发次数

**告警阈值**：
| 指标 | 警告阈值 | 严重阈值 | 检查周期 |
|------|---------|---------|---------|
| 错误率 | > 1% | > 5% | 5 分钟 |
| P0 错误 | > 0 | > 3 | 实时 |
| P1 错误 | > 3 | > 10 | 1 分钟 |
| 熔断触发 | > 5 次/小时 | > 20 次/小时 | 1 小时 |

### 4.2 通知渠道

| 错误级别 | 通知方式 | 接收人 |
|---------|---------|--------|
| P0 | 电话 + 短信 + Slack | 值班工程师 + 技术负责人 |
| P1 | 短信 + Slack | 值班工程师 |
| P2 | Slack + 邮件 | 模块负责人 |
| P3 | 工单系统 | 开发团队 |
| P4 | 工单系统 | 产品团队 |

## 5. 错误恢复流程

### 5.1 标准恢复流程

```
1. 检测错误 → 2. 记录日志 → 3. 触发告警 → 4. 执行恢复策略 → 5. 验证恢复 → 6. 关闭告警
```

### 5.2 人工介入条件

**需要人工介入的情况**：
- P0 错误持续 > 5 分钟
- 自动恢复失败 > 3 次
- 数据一致性受损
- 安全相关错误

**人工处理流程**：
1. 接收告警通知
2. 查看错误日志和监控面板
3. 定位根本原因
4. 执行修复操作（重启/回滚/数据修复）
5. 验证修复效果
6. 编写事故报告（Post-Mortem）

## 7. 错误处理检查清单

### 7.1 代码审查检查项

**错误处理**：
- [ ] 所有外部调用都有 try-catch 包裹
- [ ] 捕获特定异常，不使用裸 `except:`
- [ ] 错误日志包含完整上下文信息

## 附录 A：错误处理最佳实践

### A.1 不要做的事

❌ **不要吞掉异常**：
```python
# 错误示例
try:
    do_something()
except:
    pass  # 绝对禁止
```

❌ **不要使用过于宽泛的异常捕获**：
```python
# 错误示例
try:
    do_something()
except Exception as e:  # 应该捕获特定异常
    log.error(e)
```

❌ **不要在循环中无限制重试**：
```python
# 错误示例
while True:  # 禁止无限循环
    try:
        do_something()
        break
    except:
        pass
```

### A.2 应该做的事

✅ **捕获特定异常**：
```python
try:
    response = requests.get(url, timeout=30)
except requests.exceptions.Timeout:
    log.warning(f"请求超时：{url}")
    retry()
except requests.exceptions.ConnectionError:
    log.error(f"连接失败：{url}")
    fallback()
```

✅ **使用上下文管理器**：
```python
with open('file.txt', 'r') as f:
    content = f.read()
# 自动关闭文件
```

✅ **提供有意义的错误信息**：
```python
raise ValueError(f"用户 ID 无效：{user_id}，原因：用户不存在")
```

```
错误发生
    ↓
是否可重试？
    ├─ 是 → 执行重试（最多 3 次） → 成功？ → 结束
    │                          └─ 否 → 进入下一级
    └─ 否 → 进入下一级
         ↓
是否可降级？

是否 P0/P1 级别？

    └─ 否 → 记录日志 → 返回错误响应 → 结束
```
